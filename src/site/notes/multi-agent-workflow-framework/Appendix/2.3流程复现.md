# 2.3 流程复现

## 一、流程概览
本节展示一次完整的多Agent协作流程复现，用于清晰呈现从方案制定到脚本执行与结果审阅的全过程。通过该示例，可直观理解各角色之间的协作关系、人工参与节点以及最终产出的文档形式。

## 二、总体流程
整个流程包括：方案制定与修改、脚本编写、本地执行、一致性审阅及人工复核五个阶段，整体用时约二十分钟，其中主要耗时在人工阅读与确认环节。

## 三、方案制定阶段（数据分析Agent）
由数据分析Agent根据任务描述生成初版方案，人工多轮确认后定稿。重点是确保方案逻辑完整、输入输出明确、任务拆解合理。以下截图展示了与Agent的多轮交互过程及方案修订细节（图1–3）。

## 四、脚本编写阶段（脚本编写Agent）
脚本编写Agent根据已确认的方案生成最小可执行脚本（MVP Script），仅输出代码文本，不访问或运行真实数据。人工根据实际执行结果向Agent反馈修正意见，直至脚本结构与方案逻辑完全一致。以下截图展示脚本生成的关键过程（图4）。

脚本文档参考：[产品测试效率对比脚本](./2.3流程复现-产品测试效率对比脚本.py.md)

## 六、一致性审阅阶段（审阅Agent）
审阅Agent对方案、脚本及执行结果进行比对审查，重点验证执行逻辑是否严格遵循方案、是否存在遗漏或偏差，以及输出格式是否符合规范。以下截图展示了审阅交互过程与反馈要点（图8）。  
![审阅过程](/img/user/multi-agent-workflow-framework/Picture/2.3%E6%B5%81%E7%A8%8B%E5%A4%8D%E7%8E%B0%E5%9B%BE%E7%89%878.png)  
审阅报告参考：[脚本一致性审阅报告](./2.3流程复现-2.3_产品测试效率对比_脚本审阅.md)

## 七、人工复核与总结
人工阅读审阅报告后，对必要修改进行确认或关闭流程。至此，分析Agent制定方案、脚本Agent生成脚本、人工执行验证、审阅Agent检查一致性，再到人工最终确认，完整形成一个闭环。该流程展示了多Agent协作在分析类任务中的标准操作方式：逻辑清晰、职责分明、过程留痕、数据安全，能够为后续标准化和自动化执行提供直观参考。

## 八、结论
本次流程复现展示了多Agent在实际业务分析任务中的协作机制。通过统一的Markdown文档驱动、明确的角色分工与可追溯的交互记录，实现了高效、可控、透明的分析执行过程。这一模式可作为后续构建标准化协作框架和知识沉淀体系的基础。
